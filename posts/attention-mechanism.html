Attention Is All You Need (But Here's What That Actually Means)
Posted: 2024-10-05  |  Author: Rahul Rai  |  Tags: transformers, nlp, attention

In 2017, Vaswani et al. published "Attention Is All You Need" and proceeded to make RNNs largely obsolete. But the paper is dense and most blog posts either re-copy the abstract or drown you in matrices. Let me show you what attention actually does with real numbers.

The Core Idea

Old sequence models (LSTMs, GRUs) processed tokens one at a time — the hidden state had to carry information about the entire past. Attention says: what if every token could directly look at every other token and decide how much to "attend" to it?

The key insight: represent each token three ways —

  • Query (Q): what am I looking for?
  • Key (K): what do I offer?
  • Value (V): what is my actual content?

To compute the output for token i, you compare its query against every token's key (dot product = similarity), normalize with softmax to get weights, then take a weighted sum of all values.

The Formula

Attention(Q, K, V) = softmax( QK&sup T; / √d_k ) · V

The √d_k scaling prevents the dot products from getting too large as embedding dimension grows (which would push softmax into tiny-gradient territory).

Step by Step with Numbers

Let's do it manually. Say we have a 3-token sequence with d_model=4.

import numpy as np

# 3 tokens, d_model = 4
X = np.array([
    [1.0, 0.0, 1.0, 0.0],   # token 0
    [0.0, 1.0, 0.0, 1.0],   # token 1
    [1.0, 1.0, 0.0, 0.0],   # token 2
])

# learned projection matrices (d_model=4 → d_k=4)
np.random.seed(7)
W_Q = np.random.randn(4, 4) * 0.5
W_K = np.random.randn(4, 4) * 0.5
W_V = np.random.randn(4, 4) * 0.5

# project input to Q, K, V
Q = X @ W_Q   # shape (3, 4)
K = X @ W_K
V = X @ W_V

# compute raw attention scores
d_k = Q.shape[-1]
scores = Q @ K.T / np.sqrt(d_k)   # shape (3, 3)
print("Raw scores:\n", np.round(scores, 3))

# softmax over last axis
def softmax(x):
    e = np.exp(x - x.max(axis=-1, keepdims=True))
    return e / e.sum(axis=-1, keepdims=True)

attn_weights = softmax(scores)    # shape (3, 3)
print("\nAttention weights:\n", np.round(attn_weights, 3))

# weighted sum of values
output = attn_weights @ V         # shape (3, 4)
print("\nOutput:\n", np.round(output, 3))

Each row of attn_weights sums to 1.0 and tells you how much each token attends to every other token. The output for token 0 is a blend of all three value vectors, weighted by how relevant each token was to token 0's query.

Multi-Head Attention

A single attention head can only learn one "type" of relationship. Multi-head attention runs h attention heads in parallel, each with its own W_Q, W_K, W_V projections, then concatenates the results:

MultiHead(Q,K,V) = Concat(head₁, ..., head_h) · W_O

In GPT-style models, different heads learn different patterns — some track syntactic dependencies, others track long-range semantic relationships, some handle positional structure. 

def attention(Q, K, V):
    d_k = Q.shape[-1]
    scores = Q @ K.T / np.sqrt(d_k)
    return softmax(scores) @ V

def multi_head_attention(X, W_Qs, W_Ks, W_Vs, W_O):
    heads = [attention(X @ Wq, X @ Wk, X @ Wv)
             for Wq, Wk, Wv in zip(W_Qs, W_Ks, W_Vs)]
    concat = np.concatenate(heads, axis=-1)
    return concat @ W_O

Positional Encoding

Attention has no inherent sense of order — "cat sat mat" and "mat sat cat" would produce the same attention scores without help. Transformers add a positional encoding to each token embedding before it enters the attention layers.

The original paper used sinusoids:

PE(pos, 2i) = sin(pos / 10000^(2i/d))     PE(pos, 2i+1) = cos(pos / 10000^(2i/d))

Modern models (BERT, GPT) just learn the positional embeddings directly.

Why It Scales

The big win over RNNs: any two tokens interact in O(1) operations regardless of distance. No vanishing gradients across long sequences. The cost is O(n²) in sequence length — for a 4096-token context, that's 16M attention scores per head per layer. Techniques like Flash Attention, sliding-window attention, and sparse attention are all about making that quadratic cost manageable.

The architecture that became GPT, BERT, T5, and every LLM you use today is just this mechanism, stacked 96 times with some FFN layers in between.
