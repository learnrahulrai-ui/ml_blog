Backpropagation From Scratch: It's Just Chain Rule
Posted: 2024-11-20  |  Author: Rahul Rai  |  Tags: deep-learning, math, beginner

Every intro to neural networks gets to backprop and suddenly the notation explodes — partial derivatives, superscripts, subscripts, Jacobians. People treat it like black magic baked into PyTorch. It's not. It's one rule applied over and over: the chain rule.

The Setup

A neural network maps inputs to outputs through a chain of functions. If our network is just two layers:

ŷ = f₂( f₁( x · W₁ + b₁ ) · W₂ + b₂ )

We want to minimize a loss L(ŷ, y) by adjusting the weights. To do that we need ∂L/∂W₁, ∂L/∂W₂, etc. That's it. That's what backprop computes.

Chain Rule in One Sentence

If z = g(f(x)), then:

dz/dx = dz/df · df/dx

You have a chain of functions? Multiply all the local derivatives together. Backprop is just doing this for every parameter in the network, starting from the loss and working backwards — hence backpropagation.

A Minimal Working Example

Let's implement a 2-layer network in pure Python with no libraries. Just NumPy for matrix math. 

import numpy as np

def sigmoid(x):
    return 1 / (1 + np.exp(-x))

def sigmoid_deriv(x):
    s = sigmoid(x)
    return s * (1 - s)

def mse_loss(y_pred, y_true):
    return ((y_pred - y_true) ** 2).mean()

# --- forward pass ---
def forward(X, W1, b1, W2, b2):
    z1 = X @ W1 + b1       # pre-activation layer 1
    a1 = sigmoid(z1)       # activation layer 1
    z2 = a1 @ W2 + b2      # pre-activation layer 2
    a2 = sigmoid(z2)       # output
    cache = (X, z1, a1, z2, a2)
    return a2, cache

# --- backward pass ---
def backward(y_true, cache, W2):
    X, z1, a1, z2, a2 = cache
    m = y_true.shape[0]

    # dL/da2 — derivative of MSE loss
    dL_da2 = 2 * (a2 - y_true) / m

    # dL/dz2 — chain through sigmoid
    dL_dz2 = dL_da2 * sigmoid_deriv(z2)

    # dL/dW2 and dL/db2
    dW2 = a1.T @ dL_dz2
    db2 = dL_dz2.sum(axis=0)

    # dL/da1 — propagate gradient back through W2
    dL_da1 = dL_dz2 @ W2.T

    # dL/dz1 — chain through sigmoid again
    dL_dz1 = dL_da1 * sigmoid_deriv(z1)

    # dL/dW1 and dL/db1
    dW1 = X.T @ dL_dz1
    db1 = dL_dz1.sum(axis=0)

    return dW1, db1, dW2, db2

What's Actually Happening

Look at the backward pass line by line. We start with dL/da2 — how does the loss change as the output changes? Then at each layer we do two things:

  1. Apply the local derivative of the activation function (sigmoid_deriv) to get the gradient w.r.t. the pre-activation.
  2. Propagate through the weight matrix (@ W2.T) to get the gradient w.r.t. the previous layer's activation.

Repeat until you reach the first layer. Every gradient is just a product of gradients you already computed further down the chain. No mystery.

The Vanishing Gradient Problem (Quick Note)

Notice sigmoid_deriv(z)? The sigmoid derivative peaks at 0.25 and quickly approaches 0 for large positive or negative inputs. Multiply enough of those together as you go back through deep networks and the gradients shrink to near-zero — weights in early layers barely update. This is why ReLU (max(0, x), derivative = 1 for positive inputs) became the default activation: it keeps gradients alive.

ReLU'(x) = 1 if x > 0, else 0

Running It

# XOR dataset
X = np.array([[0,0],[0,1],[1,0],[1,1]], dtype=float)
y = np.array([[0],[1],[1],[0]], dtype=float)

# random init
np.random.seed(42)
W1 = np.random.randn(2, 4) * 0.5
b1 = np.zeros((1, 4))
W2 = np.random.randn(4, 1) * 0.5
b2 = np.zeros((1, 1))

lr = 0.5
for epoch in range(5000):
    y_pred, cache = forward(X, W1, b1, W2, b2)
    dW1, db1, dW2, db2 = backward(y, cache, W2)
    W1 -= lr * dW1
    b1 -= lr * db1
    W2 -= lr * dW2
    b2 -= lr * db2

y_pred, _ = forward(X, W1, b1, W2, b2)
print(np.round(y_pred, 2))
# → [[0.04], [0.96], [0.96], [0.04]]  ✓

Works. XOR is classically not linearly separable — the hidden layer learned to transform the input space so it becomes separable. Backprop found those weights automatically.

Takeaway

Backpropagation = forward pass to compute outputs and cache intermediate values + backward pass multiplying local gradients using the chain rule. PyTorch's autograd does exactly this, just for arbitrary computation graphs and much more efficiently. Understanding the manual version means you'll never be confused by what .backward() does.

"Backprop is just book-keeping for the chain rule." — probably someone smart
