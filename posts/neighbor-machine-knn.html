Ask the Closest Rows: Gap Formula, Same-Ruler Rule, and How Slow It Gets
Posted: 2026-06-02  |  Author: Rahul Rai  |  Tags: knn, distance, non-parametric, complexity

The ask-closest rule guesses by averaging the k closest working-pile right-answers. Nothing is built ahead of time — the whole pile is stored and looked up each time. Three steps — measure gap, pick closest, average — but two details decide whether it works: put columns on the same ruler, and pick k with care.

Working Pile and Hidden Pile

   WORKING PILE (rows with right answers)   HIDDEN PILE (right answers withheld)
   rooms income age pop │ price             rooms income age pop │ price
   ─────────────────────────────            ─────────────────────────────
     4    3.2   6   320 │  3.4               3    3.0   5  410 │  ?
     8    1.5   1   880 │  1.5              ...                 │  ?

Gaps are measured from the measured columns only. Hidden right-answers are read once, at scoring — never during guessing.

Measuring the Gap

Straight-line gap over all measured columns:

gap(a, b) = √( Σj (aj − bj)² )

This is p = 2 of the general rule (Σj |aj − bj|p)1/p. At p = 1 it counts straight city-block steps. Worked pair — new row (3, 2, 5, 4): 

   row A: (4, 2, 6, 3)   gaps (1, 0, 1, 1)   gap = √(1+0+1+1)    = √3   ≈ 1.73
   row B: (8, 9, 1, 9)   gaps (5, 7, 4, 5)   gap = √(25+49+16+25) = √115 ≈ 10.72

Put Columns on the Same Ruler First

The gap adds squared per-column differences. A column measured in thousands (people) drowns one measured in single digits (rooms) — an accident of units, not importance. On the raw California pile, the ask-closest rule is effectively a 1-column rule on population. 

   raw columns:
     rooms  ≈   2 – 12     ← tiny gaps
     people ≈ 100 – 35000  ← huge gaps  ← this column runs the whole show

Fix: shift each column to zero average and stretch it to spread of 1, using numbers from the working pile only:

zj = (xj − μj) / σj

Compute μj, σj on the working pile; apply the same constants to the hidden pile. After this, every column contributes equally to the gap. (Squeezing to [0,1] instead of unit spread is an alternative that works the same way.)

⚠ LEARN THE RULER ON THE WORKING PILE ONLY Computing averages and spreads on the whole pile before splitting leaks hidden-pile numbers into the build step. Learn on working, apply to both. Inside rotating folds, relearn per round — the bound pipeline does this.

Guessing One Row

   1. measure gap to every working-pile row          O(n·d)
   2. pick the k smallest gaps                        O(n) with a heap
   3. average those k right-answers  →  guess

An optional refinement weights closer rows more — inverse-gap weights wi = 1/gapi:

guess = ( Σi wi yi ) / ( Σi wi )

Equal weights (wi = 1) recover the plain average.

How Slow Is It

No work done ahead of time: the pile is stored, not compressed. Every guess recomputes all gaps from scratch. 

   100 rows = 80 working + 20 hidden
   work per hidden row  = 80 × d  gap computations
   total work           = 20 × 80 × d

Sorting shelves (KD-tree, Ball-tree) cut average guess time to roughly O(d·log n) when columns are few, but degrade toward brute force as columns pile up. Here is why: 

   few columns:   nearby rows are truly nearby  ← sorting shelf helps
   many columns:  all rows sit about the same gap away
                  "closest" stops meaning anything

Worked by Hand

One column (income) only, so same-ruler step is moot here. New row at income 5.0, k = 3: 

   working pile (income, price):
     (5.6, 3.4)  (4.0, 2.7)  (3.8, 2.4)  (2.5, 1.5)  (7.3, 3.5)

   gaps from 5.0:
     |5.0 − 5.6| = 0.6 ←   |5.0 − 4.0| = 1.0 ←   |5.0 − 3.8| = 1.2 ←
     |5.0 − 2.5| = 2.5      |5.0 − 7.3| = 2.3

   3 closest right-answers: 3.4, 2.7, 2.4
   guess = (3.4 + 2.7 + 2.4) / 3 = 2.83

In the Toolbox

Binding the ruler-learner to the ask-closest rule means the ruler is relearned on each round's building portion — never on the grading chunk: 

from sklearn.pipeline import make_pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.neighbors import KNeighborsRegressor
from sklearn.metrics import mean_squared_error

knn = make_pipeline(StandardScaler(), KNeighborsRegressor(n_neighbors=5))
knn.fit(X_train, y_train)          # stores pile; learns μ, σ on working rows
pred = knn.predict(X_test)         # same ruler → measure gaps → average

rmse = mean_squared_error(y_test, pred) ** 0.5
print(round(rmse, 3))

k Is the Stiff-vs-Jumpy Dial

   k = 1       asks 1 row     → copies its noise exactly
               zero mistake on working pile
               bad on new rows   (too jumpy)

   k = large   asks many rows → smooths everything out
               always a bit wrong, everywhere  (too stiff)
               at k = n_working, every guess = average price

   right k  →  somewhere in between — found by rotating folds

Picking k Without Touching the Hidden Pile

Try each candidate k by rotating folds on the working pile. Passing the whole bound rule to the rotating-fold checker means the ruler is relearned per round — no peeking: 

from sklearn.model_selection import KFold, cross_val_score

def fold_mistake(k, X_train, y_train):
    rule = make_pipeline(StandardScaler(), KNeighborsRegressor(n_neighbors=k))
    kf = KFold(n_splits=5, shuffle=True, random_state=RANDOM_STATE)
    scores = cross_val_score(rule, X_train, y_train, cv=kf,
                             scoring='neg_root_mean_squared_error')
    return round(-scores.mean(), 3)

candidates = [1, 3, 5, 7, 9, 15, 25]
cv = {k: fold_mistake(k, X_train, y_train) for k in candidates}
best_k = min(cv, key=cv.get)    # key with the smallest VALUE
⚠ THREE THINGS THAT TRIP PEOPLE UP (1) k (how many neighbours) and the fold count (how many grading rounds) are unrelated — k=25 with 5 folds is fine.
(2) The toolbox maximises its score, so it negates the mistake; un-negate with -scores.mean().
(3) min(cv, key=cv.get) returns the key with the smallest value; plain min(cv) returns the smallest key — wrong answer.

Origin

Evelyn Fix and Joseph Hodges wrote down this rule in 1951. Thomas Cover and Peter Hart proved in 1967 that the one-closest-row version's mistake rate is at most twice the best possible mistake, as the pile grows without bound.

  1. Guess = average of the k closest working right-answers (optionally weighted by closeness).
  2. The gap adds squared per-column differences — put columns on the same ruler, learning it on the working pile only.
  3. No building ahead; each guess costs O(n·d) and stores the whole pile.
  4. k sets the stiff-vs-jumpy balance; pick it by rotating folds.
  5. When columns pile up, close stops meaning close.
