k-Nearest Neighbours: Derivation, Scaling, and Complexity
Posted: 2026-06-02  |  Author: Rahul Rai  |  Tags: knn, distance, non-parametric, complexity

KNN predicts a target by averaging the k training targets closest in feature space. It is non-parametric and instance-based: there is no fit step beyond storing the data. The whole method is three operations — distance, select, average — but two practical details decide whether it works at all: feature scaling and the choice of k.

Train and Test

   TRAIN (n rows, labeled)             TEST (targets withheld)
   rooms income age pop │ price        rooms income age pop │ price
   ──────────────────────────────      ──────────────────────────────
     4    3.2   6   320 │  3.4           3    3.0   5  410 │  ?
     8    1.5   1   880 │  1.5           ...                │  ?

Distances are computed from features only. The test targets are read once, at scoring — never during prediction.

Distance

Euclidean distance over all features:

d(a, b) = √( Σj (aj − bj)² )

This is the p = 2 case of the Minkowski metric (Σj |aj − bj|p)1/p; p = 1 is Manhattan distance. A worked pair, test row (3, 2, 5, 4): 

   row A: (4, 2, 6, 3)  gaps (1,0,1,1)  d = √(1+0+1+1)   = √3   ≈ 1.73
   row B: (8, 9, 1, 9)  gaps (5,7,4,5)  d = √(25+49+16+25) = √115 ≈ 10.72

Scale First, or the Distance Lies

The distance sums squared per-feature gaps, so a feature measured in thousands (population) swamps one measured in single digits (rooms). Whichever feature has the largest numeric range dominates — usually an accident of units, not a statement about importance. On the raw California features, KNN is effectively a 1-D model on population.

Fix it by standardising each feature to zero mean and unit variance, using statistics computed on the training set only:

zj = (xj − μj) / σj

Fit μj, σj on train; apply the same constants to test. (Min–max scaling to [0,1] is an alternative.) After scaling, every feature contributes comparably to the distance.

⚠ FIT THE SCALER ON TRAIN ONLY Computing μ, σ over the full dataset before splitting leaks test statistics into training — the model has seen a summary of the test set. Standardise after the split, fitting on train, transforming both. Inside cross-validation, the scaler must be refit per fold (use a Pipeline).

Predicting One Row

   1. distance to all n training rows        O(n·d)
   2. select the k smallest                  O(n) with a partial sort / heap
   3. average those k targets  →  ŷ

For regression the prediction is the mean of the k neighbour targets. An optional refinement weights nearer neighbours more heavily — inverse-distance weights wi = 1/di:

ŷ = ( Σi wi yi ) / ( Σi wi )

Uniform weights (wi = 1) recover the plain mean.

Cost

Brute force computes n_train distances per query at O(d) each — O(n_train · d) per query, O(n_train · n_query · d) total. There is no training phase to amortise this: KNN trades a zero-cost fit for an expensive predict, and stores the entire dataset. 

   100 rows = 80 train + 20 query
   work     = 20 × 80 × d  distance evaluations

Spatial indexes (KD-tree, Ball-tree) cut average query time to roughly O(d·log n) in low dimensions, but degrade toward brute force as d grows. This is the curse of dimensionality: in high dimensions, distances concentrate — the nearest and farthest points sit at almost the same distance — and "nearest neighbour" stops being meaningful.

Worked by Hand

One feature (income), so scaling is moot here; new row at income 5.0, k = 3: 

   train (income, price):
     (5.6, 3.4) (4.0, 2.7) (3.8, 2.4) (2.5, 1.5) (7.3, 3.5)

   |5.0 − 5.6| = 0.6   |5.0 − 4.0| = 1.0   |5.0 − 3.8| = 1.2
   |5.0 − 2.5| = 2.5   |5.0 − 7.3| = 2.3

   3 nearest prices: 3.4, 2.7, 2.4
   ŷ = (3.4 + 2.7 + 2.4) / 3 = 2.83

In Code

A Pipeline binds the scaler to the estimator so scaling is fit on train and reapplied on predict — and, later, refit per CV fold: 

from sklearn.pipeline import make_pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.neighbors import KNeighborsRegressor
from sklearn.metrics import mean_squared_error

knn = make_pipeline(StandardScaler(), KNeighborsRegressor(n_neighbors=5))
knn.fit(X_train, y_train)                 # store data; scaler learns μ, σ on train
pred = knn.predict(X_test)                # scale → distance → select → average

rmse = mean_squared_error(y_test, pred) ** 0.5
print(round(rmse, 3))

k Is the Bias–Variance Knob

k is the only real hyperparameter (alongside the metric and the weighting scheme). It controls model complexity directly:

  • k = 1: zero training error, the prediction surface fits every noisy point — low bias, high variance.
  • k large: each prediction averages many points and flattens toward the global mean — high bias, low variance. At k = n_train, every query returns ȳ.

Pick k by cross-validation, not by eye.

Choosing k Without Touching the Test Set

Cross-validate each candidate on the training set. Passing the whole Pipeline to cross_val_score refits the scaler on each fold's training portion — no leakage: 

from sklearn.model_selection import KFold, cross_val_score

def cv_rmse(k, X_train, y_train):
    model = make_pipeline(StandardScaler(), KNeighborsRegressor(n_neighbors=k))
    kf = KFold(n_splits=5, shuffle=True, random_state=RANDOM_STATE)
    scores = cross_val_score(model, X_train, y_train, cv=kf,
                             scoring='neg_root_mean_squared_error')
    return round(-scores.mean(), 3)

candidates = [1, 3, 5, 7, 9, 15, 25]
cv = {k: cv_rmse(k, X_train, y_train) for k in candidates}
best_k = min(cv, key=cv.get)              # key with the smallest VALUE
⚠ THREE THINGS THAT TRIP PEOPLE UP (1) k (neighbours, sets the guess) and the fold count (grading, sets the estimate) are unrelated — k=25 with 5 folds is fine.
(2) scikit-learn maximises its score, so RMSE is negated; un-negate with -scores.mean().
(3) min(cv, key=cv.get) returns the key with the smallest value; plain min(cv) returns the smallest key — the wrong answer.

Origin

The rule was formalised by Evelyn Fix and Joseph Hodges in 1951 and analysed by Thomas Cover and Peter Hart in 1967, who proved the 1-NN error rate is asymptotically within a factor of two of the optimal Bayes error.

  1. Prediction = mean of the k nearest training targets (optionally distance-weighted).
  2. Euclidean distance is scale-sensitive — standardise features, fitting on train only.
  3. No training; predict is O(n_train·d) per query and stores all data.
  4. k sets the bias–variance trade-off; choose it by cross-validation.
  5. High dimensions erode the method (distance concentration).
