Linear Regression: OLS, the Normal Equations, and Numerics
Posted: 2026-06-01  |  Author: Rahul Rai  |  Tags: linear-regression, ols, least-squares, numerical-stability

Linear regression models the target as a weighted sum of the features plus an intercept. Unlike KNN it is parametric: fit once, discard the data, keep d+1 numbers. The weights have a closed-form least-squares solution.

The Model

ŷi = β0 + Σj βj xij   ⇔   ŷ = Xβ

The intercept β0 is absorbed by prepending a column of ones to X. Each βj is the change in the prediction per unit change in feature j, holding the others fixed. The model is linear in the parameters; the features themselves may be transformed (squares, logs, interactions) and it remains linear regression.

The Objective: Least Squares

Choose β to minimise the residual sum of squares:

J(β) = ‖Xβ − y‖² = Σi (ŷi − yi)²

J is a convex quadratic — its Hessian 2XᵀX is positive semidefinite — so any stationary point is a global minimum. No local minima, no iterative search required. 

         ╲         ╱
          ╲       ╱        height  = squared error J(β)
           ╲     ╱         bottom  = the least-squares β
            ╲ • ╱

Solving It: The Normal Equations

Set the gradient to zero:

∇J = 2Xᵀ(Xβ − y) = 0  ⇒  XᵀXβ = Xᵀy  ⇒  β = (XᵀX)−1Xᵀy

The same system, solved by hand at d = 1 (one feature, one intercept) on three points — here Σ1, Σx etc. are the entries of XᵀX and Xᵀy: 

   x = 1, 2, 3     y = 2, 3, 5
   Σ1 = 3   Σx = 6   Σy = 10   Σxy = 23   Σx² = 14

   XᵀX β = Xᵀy   →    3b +  6w = 10
                      6b + 14w = 23

   solve:  2w = 3  →  w = 3/2 ;   3b + 9 = 10  →  b = 1/3
   line:   ŷ = 1/3 + (3/2)x

Why the Residuals Check Out

At the optimum, Xᵀ(y − Xβ) = 0, i.e. Xᵀe = 0: the residual vector is orthogonal to every feature column, and (because one column is all ones) the residuals sum to zero. That's exactly what the three-point fit shows: 

   x=1: ŷ=1.833, r = y−ŷ = +0.167
   x=2: ŷ=3.333, r       = −0.333
   x=3: ŷ=4.833, r       = +0.167

   Σ r      = 0     ✓   (orthogonal to the intercept column)
   Σ x·r    = 0     ✓   (orthogonal to the x column)
   RMSE = √(mean r²) ≈ 0.236

Geometrically, ŷ = Hy with the hat matrix H = X(XᵀX)−1Xᵀ — the orthogonal projection of y onto the column space of X. H is symmetric and idempotent (H² = H), and trace(H) equals the number of fitted parameters. OLS is that projection; no vector in the span of the features sits closer to y.

★ SAME OPERATION, SCALED UP The three-point pencil solve is identical to what the library runs on 16,000 rows and 8 features — only the dimensions of XᵀX change. Fitted weights land in m.coef_, the intercept in m.intercept_.

When OLS Is the Best Linear Estimator

The Gauss–Markov theorem: if the errors have zero conditional mean (E[ε|X] = 0), constant variance, and are uncorrelated, and X has full column rank, then OLS is BLUE — the minimum-variance estimator among all linear unbiased estimators. Two things worth stating precisely:

  • Normality is not required for Gauss–Markov. You only need it for exact confidence intervals and t-tests, not for OLS to be the best linear unbiased fit.
  • Full column rank means no feature is a linear combination of the others. If one is, XᵀX is singular and (XᵀX)−1 does not exist.

Numerics: Don't Invert the Matrix

Forming XᵀX costs O(nd²) and solving costs O(d³), so the fit is O(nd² + d³). But you should not compute (XᵀX)−1 explicitly: forming XᵀX squares the condition number (κ(XᵀX) = κ(X)²) and loses precision. Instead:

  • Solve the linear system directly (Cholesky on XᵀX), or better,
  • Factor X itself with QR or SVD and solve the least-squares problem without ever forming XᵀX. scikit-learn's LinearRegression uses an SVD-based solver.
  • For very large n or d, or streaming data, use gradient descent: O(nd) per step instead of a full factorisation.
  • If XᵀX is singular or ill-conditioned (collinear features), add an L2 penalty — ridge regression, β = (XᵀX + λI)−1Xᵀy — which is invertible for any λ > 0.
▸ SCALING: THE CONTRAST WITH KNN Plain OLS is invariant to feature scaling: rescale a feature and its weight rescales inversely, leaving predictions and RMSE unchanged. So — unlike KNN — OLS needs no standardisation for correctness. You do need it for gradient-descent convergence and for any regularised variant (ridge/lasso penalise coefficient magnitude, which depends on scale).

In Code

from sklearn.linear_model import LinearRegression
from sklearn.metrics import mean_squared_error

m = LinearRegression()           # SVD-based least squares under the hood
m.fit(X_train, y_train)          # solves XᵀX β = Xᵀy for β
pred = m.predict(X_test)         # one dot product per row

rmse = mean_squared_error(y_test, pred) ** 0.5
print(round(rmse, 3), m.coef_, m.intercept_)

KNN vs. OLS

 KNNLinear regression (OLS)
Typenon-parametric, instance-basedparametric
Fitstore data, O(1)solve normal equations, O(nd²+d³)
PredictO(n·d) per rowO(d) per row
Hyperparameterk (cross-validate)none (λ if regularised)
Feature scalingrequirednot for plain OLS
Hypothesislocal, piecewise-constantone global hyperplane

Opposite strategies, one question: given the rows seen, predict the target for a row unseen. KNN leans on stored examples; OLS compresses them into a hyperplane.
