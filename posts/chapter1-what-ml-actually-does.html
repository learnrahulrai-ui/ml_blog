Supervised Regression, End to End (No Jargon Until It's Earned)
Posted: 2026-06-03  |  Author: Rahul Rai  |  Tags: fundamentals, supervised-learning, knn, linear-regression

Most ML explanations define jargon with more jargon. This series builds the machinery first — by hand, on the California housing dataset — and attaches the standard terms at the end. Part 1 is the whole supervised-regression pipeline; Parts 2 and 3 derive the two estimators in detail.

The Problem: Learn a Function From a Table

You're given a table of n rows and d feature columns — a matrix X ∈ ℝn×d — plus a target vector y ∈ ℝn (here, median house value per district). The goal: learn a function f: ℝd → ℝ that minimises prediction error on rows it has never seen, not on the rows it was built from. 

   rooms  income  age  ocean  people  │  PRICE  (= target y)
   ──────────────────────────────────────────────────────
    4.2    3.5    25    1.0    320    │   1.4
    5.1    5.6    10    0.5    210    │   2.8
    3.8    2.1    42    3.2    890    │   0.9
   └──────────── features X ─────────┘

California housing is shape (20640, 9) — 20,640 rows, 8 features plus the target. The features sit on different numeric scales (median income ≈ 0–15, population ≈ hundreds to thousands). That difference is harmless for a linear model but breaks distance-based methods unless you rescale — see Part 2.

⚠ .shape is a PAIR, not a product df.shape returns the tuple (n_rows, n_cols) in that fixed order — (20640, 9) — not n_rows×n_cols.

The One Rule: Isolate a Test Set

Before fitting anything, hold out a random fraction of the rows (20% here). Their targets must not influence any decision — not model choice, not hyperparameters, not even the parameters of a preprocessing step. They exist to produce one unbiased estimate of generalisation error, read once, at the very end. 

   ████████████████░░░░
    80% train          20% test (read once)

The reason is mechanical, not moral: any quantity fit on a set and then evaluated on that same set is optimistically biased. Training error underestimates test error by construction — at k=1, nearest-neighbours scores a perfect 0 on its own training data and generalises poorly. The held-out set is the only honest estimate of performance on the population the data was drawn from (assuming the rows are roughly i.i.d.).

★ SPLIT DISCIPLINE Shuffle before splitting (datasets often arrive sorted) with a fixed random seed so the partition is reproducible across runs. The seed controls reproducibility, not fairness — the shuffle is what removes ordering bias.

Estimator 1 — k-Nearest Neighbours (Instance-Based)

Non-parametric: nothing is fit; the training set is the model. To predict a row, compute its distance to every training row, take the k closest, and return the mean of their targets.

d(a, b) = √( Σj (aj − bj)² )

Because the distance sums over features, any feature with a large numeric range dominates it — so KNN requires feature scaling. The neighbour count k is a hyperparameter trading bias against variance: small k → low bias, high variance; large k → high bias, low variance. Full derivation, scaling, and complexity in Part 2.

Scoring: RMSE

Define the residual ri = yi − ŷi (truth minus prediction). The root-mean-squared error is:

RMSE = √( (1/n) Σi ri² )

RMSE carries the target's units, so an RMSE of 0.5 here is a typical miss of about $50,000 (the target is in $100,000s). The square makes it more sensitive to large errors than the mean absolute error, MAE = mean|ri|. The reference point: predicting the constant ȳ for every row gives RMSE = std(y) — any useful model must beat that baseline.

Estimator 2 — Linear Regression (Parametric)

A parametric model with d+1 numbers (one weight per feature, plus an intercept):

ŷ = Xβ   (weights β, intercept folded in as a constant column)

Fit by ordinary least squares: choose β minimising the squared residual norm ‖Xβ − y‖². For a linear model this objective is convex with a closed-form solution — no iterative search. Once fitted, prediction is a single dot product; the training data is discarded. Derivation, assumptions, and numerics in Part 3.

Model Selection: k-Fold Cross-Validation

You can't tune k on the test set without contaminating it. Instead, partition the training set into K folds; for each candidate setting, train on K−1 folds, score on the held-out fold, rotate so every fold is held out exactly once, and average the scores. 

            A    B    C    D    E
   round 1  ⬛   ·    ·    ·    ·    → RMSE₁
   round 2  ·    ⬛   ·    ·    ·    → RMSE₂
   round 3  ·    ·    ⬛   ·    ·    → RMSE₃
   round 4  ·    ·    ·    ⬛   ·    → RMSE₄
   round 5  ·    ·    ·    ·    ⬛   → RMSE₅
                              mean → CV estimate of generalisation error
⚠ PREPROCESSING GOES INSIDE THE FOLD Any step with learned parameters (a scaler's mean/variance, an imputer's fill value) must be fit on each fold's training portion only. Fitting it on the full training set before splitting leaks information across folds and biases the estimate optimistically. A scikit-learn Pipeline enforces this automatically (Part 2).

Procedure: select hyperparameters by CV on the training set → refit the winner on all of training → evaluate once on the test set → report.

▸ "Isn't scoring on the test set peeking?" No — the invariant is never let the test set influence a choice. Using it to select among models is leakage; using it once to estimate the chosen model's error is its defined purpose. A test RMSE modestly above the CV RMSE is expected (CV slightly overfits the selection); a large gap signals overfitting or distribution shift.

The Pipeline in Six Lines

  1. Learn f: X → y that generalises; measure on held-out data only.
  2. KNN — non-parametric, distance-based; scale features, tune k.
  3. Linear regression — parametric, OLS, closed-form weights.
  4. Score with RMSE (target units); the bar to clear is std(y).
  5. Tune by k-fold CV with preprocessing fit inside each fold.
  6. Refit the winner on all training data; read the test set exactly once.
