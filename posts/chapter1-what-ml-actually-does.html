Guessing House Prices, End to End (No Labels Until the End)
Posted: 2026-06-03  |  Author: Rahul Rai  |  Tags: fundamentals, supervised-learning, knn, linear-regression

Most explanations define labels with more labels. This series builds the machinery first — by hand, on the California housing pile — and sticks the standard labels on at the very end. Part 1 is the whole picture; Parts 2 and 3 work out each rule in detail.

The Job: Guess From a Sheet of Numbers

  rooms  income  age  ocean  people  │  PRICE  ← right-answer column
  ─────────────────────────────────────────────────
   4.2    3.5    25    1.0    320    │   1.4
   5.1    5.6    10    0.5    210    │   2.8
   3.8    2.1    42    3.2    890    │   0.9
  └───────── 8 measured columns ────┘

20,640 rows. Each row is one California neighbourhood. 8 measured columns. One right-answer column: median house price in $100,000s. The job: build a rule that guesses the right answer for a row it has never seen, not a row it learned from.

⚠ .shape is a PAIR, not a product df.shape returns (n_rows, n_cols) in that fixed order — (20640, 9) — never their product.

The One Rule: Hide Some Rows

   ALL ROWS
   ████████████████░░░░
    working pile (80%)   hidden pile (20%)
                         ↑ locked away — read once, at the very end

Before touching anything: set aside a random 20% of rows. Their right answers must not touch any decision — not the rule, not any setting, not even how the numbers are prepared. They exist for one honest check at the end.

The reason is mechanical: any rule checked on the same rows it was built from reports a false low mistake. The ask-closest rule at k=1 makes zero mistakes on its own working pile and does poorly on new rows. The hidden pile is the only honest number.

★ SHUFFLE FIRST Rows often arrive sorted by district. Shuffle with a fixed starting point so the split is reproducible and not ordered. The fixed point controls reproducibility; the shuffle removes order bias.

Rule 1 — Ask the Closest Rows

   new row ?
       ↓
   measure gap to every working-pile row
       ↓
   pick the k closest
       ↓
   average their right-answers   →   guess
gap(a, b) = √( Σj (aj − bj)² )

Nothing is built ahead of time — the whole working pile is stored and looked up each time. The one setting to pick by hand: k (how many neighbours to ask). Small k → too jumpy; large k → too stiff. Full derivation in Part 2.

Measuring How Wrong: RMSE

Leftover for one row: ri = yi − ŷi (right answer minus guess).

RMSE = √( (1/n) Σi ri² )

RMSE stays in the same units as the right-answer column. RMSE = 0.5 here means a typical miss of $50,000. The bar to beat: always guessing the average price gives RMSE = std(y). Any useful rule must beat that. RMSE punishes large single misses more heavily than the plain average of absolute leftovers, MAE = mean|ri|.

Rule 2 — The Straight-Stick Rule

   d+1 dials:  [nudge β₀,  dial₁ β₁,  dial₂ β₂, …]

   guess = β₀ + β₁·(rooms) + β₂·(income) + …
         = Xβ   (one dot-product per row)

Squeeze the whole working pile into d+1 dials. Once set, the pile is discarded; each guess is one dot-product. The dials are set by shrinking the squared leftover sum — an exact formula, no searching. Full derivation in Part 3.

Picking Settings Without Peeking at the Hidden Pile

   working pile split into 5 chunks:  A  B  C  D  E

   round 1  ⬛   ·    ·    ·    ·    → mistake₁
   round 2  ·    ⬛   ·    ·    ·    → mistake₂
   round 3  ·    ·    ⬛   ·    ·    → mistake₃
   round 4  ·    ·    ·    ⬛   ·    → mistake₄
   round 5  ·    ·    ·    ·    ⬛   → mistake₅
                              average → honest estimate

Each round, one chunk grades the rule built on the other four. Rotate until every chunk has graded once. Average the five mistakes. This gives an honest estimate without ever touching the hidden pile.

⚠ PREPARE NUMBERS INSIDE EACH ROUND Any step that learns from the numbers (computing an average, a spread) must be done on each round's building portion only. Doing it on the whole working pile before splitting leaks grading-chunk numbers into the build step. A bound pipeline enforces this automatically (Part 2).

Procedure: pick settings using rotating folds on the working pile → refit the winner on all of the working pile → read the hidden pile once → report that number.

▸ Isn't checking the hidden pile peeking? No — using it to pick among rules is peeking; using it once to report the chosen rule's mistake is exactly what it is for. A gap between the rotating-fold estimate and the hidden-pile mistake means the rule memorised the working pile a little.

The Six Steps

  1. Find a rule that guesses right on rows never seen; measure on the hidden pile only.
  2. Ask-closest rule — store the pile, look it up; put columns on the same ruler; pick k.
  3. Straight-stick rule — d+1 dials, exact formula, one dot-product per guess.
  4. Measure mistake with RMSE (same units as price); the bar to beat is std(y).
  5. Pick settings by rotating folds, preparing numbers inside each round.
  6. Refit on all of working pile; read the hidden pile exactly once.
